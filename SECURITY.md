# Seguridad

## Reporte responsable

No publiques detalles explotables ni errores clínicos potencialmente peligrosos en una discusión abierta. Comunícalos al propietario del repositorio y marca el reporte como crítico si puede afectar dosis, energía o secuencia terapéutica.

## Datos

La aplicación no requiere cuentas ni servidor. No deben registrarse datos clínicos identificables. El estado de simulación permanece en el almacenamiento local del navegador hasta que se borra.

## Dependencias

La aplicación en producción no usa dependencias de JavaScript externas. Las acciones de GitHub deben revisarse y actualizarse de forma periódica.
