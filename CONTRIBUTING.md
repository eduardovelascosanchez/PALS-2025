# Contribuir

1. Abre una rama desde `main`.
2. Mantén el contenido clínico en `assets/js/data.js` y la lógica matemática en `assets/js/clinical.js`.
3. Añade pruebas cuando cambie una fórmula, un límite o una ventana temporal.
4. Ejecuta `npm run check` antes de abrir el pull request.
5. Describe fuente, fecha, impacto clínico y prueba realizada.
6. Solicita revisión conforme a `docs/CLINICAL_GOVERNANCE.md`.

No incluyas datos de pacientes, claves, materiales de curso restringidos, logotipos o diagramas de terceros sin autorización.
